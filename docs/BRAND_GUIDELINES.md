## 2. Visual Identity

### 2.1 Logo System

**Primary Logo:** RECTOR LABS "R" symbol
- Composed of tech/science iconography (microscope, DNA, circuits, atoms, gears, lightbulb, etc.)
- Forms the letter "R"
- Represents intersection of science, technology, and innovation

**Logo Variations:**

1. **Logo Icon Only** (`rectorlabs_logo_3.png`)
   - Just the "R" symbol
   - Use: Favicons, app icons, social media avatars

2. **Logo with Text - Light** (`rectorlabs_logo_1.png`)
   - "R" symbol on white/light background
   - Use: Light themed applications, print materials

3. **Logo with Text - Dark** (`rectorlabs_logo_2.png`)
   - "R" symbol + "RECTOR LABS" text on dark navy background
   - Use: Dark themed applications, primary digital usage

**Logo Usage Rules:**
- ✅ Maintain aspect ratio always
- ✅ Ensure sufficient whitespace around logo
- ✅ Use provided color variations
- ❌ Do not distort or skew
- ❌ Do not change colors
- ❌ Do not add effects (shadows, gradients on logo itself)
- ❌ Do not place on busy backgrounds

**Minimum Clear Space:**
- Maintain space equal to the height of "R" symbol on all sides

**Minimum Size:**
- Digital: 32px height
- Print: 0.5 inches height

### 2.3 Typography

#### Font Families

**Headings:** Space Grotesk
- Weights: Medium (500), Semi-bold (600), Bold (700)
- Character: Modern, geometric, technical
- Usage: All headings (H1-H6), hero text, titles
- Fallback: `'Space Grotesk', 'Inter', sans-serif`

**Body Text:** Inter
- Weights: Light (300), Regular (400), Medium (500), Semi-bold (600), Bold (700)
- Character: Clean, readable, professional
- Usage: Paragraphs, UI elements, general content
- Fallback: `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`

**Monospace/Code:** JetBrains Mono
- Weights: Regular (400), Medium (500), Bold (700)
- Character: Developer-focused, clear
- Usage: Code blocks, cheatsheets, technical content
- Fallback: `'JetBrains Mono', 'Fira Code', 'Consolas', monospace`

- H1: 2rem (32px)
- H2: 1.75rem (28px)
- H3: 1.5rem (24px)

**Tablet (768px - 1024px):**
- Hero: 3rem (48px)
- H1: 2.5rem (40px)
- H2: 2rem (32px)
- H3: 1.75rem (28px)

**Desktop (>1024px):**
- Use full scale as defined above

#### Typography Best Practices

- **Line Length:** 60-75 characters for optimal readability
- **Paragraph Spacing:** 1.5x line height
- **Letter Spacing:** Use default for most text; slight increase for all-caps
- **Hierarchy:** Maintain clear visual hierarchy with size, weight, color

## 3. Voice & Tone

### 3.1 Brand Voice

**Core Attributes:**
- **Professional yet approachable:** Technical but not intimidating
- **Honest and transparent:** No hype, no exaggeration
- **Purpose-driven:** Emphasize value and impact
- **Islamic-aware:** Natural integration of Islamic expressions
- **Concise:** Respect reader's time

### 3.2 Writing Guidelines

**Do:**
- Use active voice
- Be specific and concrete
- Use technical terms when appropriate
- Include Islamic expressions naturally (1-2 per section)
- Show, don't just tell

**Don't:**
- Use corporate jargon or buzzwords
- Overcomplicate explanations
- Make false promises
- Use excessive emojis (unless contextually appropriate)
- Overuse superlatives

**Islamic Expressions Usage:**
See global CLAUDE.md for comprehensive list. Examples:
- Bismillah (beginning tasks)
- Alhamdulillah (gratitude)
- InshaAllah (future plans)
- MashaAllah (appreciation)

## 4. UI Components

### 4.1 Buttons

**Primary Button:**
- Background: Brand gradient
- Text: White, Semi-bold
- Padding: `md` (vertical) × `xl` (horizontal)
- Border-radius: 8px
- Hover: Add subtle glow or brightness increase

**Secondary Button:**
- Background: Transparent
- Border: 2px solid brand blue
- Text: Brand blue, Semi-bold
- Hover: Fill with brand gradient

**Text Button:**
- Background: None
- Text: Brand blue, Medium weight
- Hover: Underline + color shift to turquoise

### 4.2 Cards

**Style:**
- Background: White (light mode), `#2C3E50` (dark mode)
- Border: None or subtle `1px solid #E0E0E0`
- Border-radius: 12px
- Shadow: Subtle elevation
- Padding: `lg` to `xl`

**Card Hover:**
- Slight elevation increase
- Border color: Brand gradient
- Transform: `translateY(-4px)`

### 4.3 Forms

**Input Fields:**
- Border: `1px solid #E0E0E0`
- Border-radius: 8px
- Padding: `md`
- Focus: Border color → brand blue, add glow

**Labels:**
- Font: Inter, Medium (500)
- Size: Small (14px)
- Color: `#2C3E50` (light), `#FFFFFF` (dark)

**Validation:**
- Success: Green border + check icon
- Error: Red border + error icon + message
- Info: Blue border + info icon

### 4.4 Navigation

**Header:**
- Background: Dark navy (preferred) or white
- Height: 64px
- Logo: Left aligned
- Links: Right aligned, horizontal
- Mobile: Hamburger menu

**Footer:**
- Background: Dark navy
- Text: White or light gray
- Links: Turquoise on hover
- Sections: About, Projects, Contact, Social

## 6. Accessibility

### 6.1 Requirements

- ✅ WCAG 2.1 AA compliance (minimum)
- ✅ Keyboard navigation support
- ✅ Screen reader compatibility
- ✅ Focus indicators visible
- ✅ Sufficient color contrast (4.5:1 text, 3:1 UI)
- ✅ Alt text for all images
- ✅ Semantic HTML

### 6.2 Best Practices

- Use proper heading hierarchy (H1 → H2 → H3)
- Provide skip links for navigation
- Ensure forms have proper labels
- Use ARIA labels where needed
- Test with screen readers
- Keyboard-only navigation testing

## 7. Animation & Motion

### 7.1 Principles

- **Purposeful:** Animations should enhance UX, not distract
- **Fast:** Transitions <300ms for most interactions
- **Natural:** Use easing functions (ease-in-out)
- **Respectful:** Respect prefers-reduced-motion

### 7.2 Common Animations

**Page Transitions:**
- Fade in: 200ms
- Slide in: 300ms

**Hover Effects:**
- Color change: 150ms
- Scale: 200ms
- Elevation: 150ms

**Loading:**
- Skeleton screens (preferred)
- Spinners with brand colors

**Scroll Animations:**
- Fade in as elements enter viewport
- Subtle parallax for hero sections
